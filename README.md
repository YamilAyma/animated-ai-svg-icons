## 🚀 Key Features

* **Hardware-Accelerated CSS:** Animations are written directly in the `<style>` tag of each SVG using high-performance CSS `transform` properties and efficient cubic-bezier transitions. No bulky JavaScript required!
* **Fully Scalable & Responsive:** Since they are native SVGs, they scale beautifully to any size (from a `16x16` browser favicon to a huge hero background illustration) without any pixelation or performance loss.
* **Pastel & Modern Palettes:** Calibrated colors featuring soft purple, orange, blue, and yellow hues with thick organic outlines that look incredibly cohesive when used together.
* **Perpetual Seamless Loops:** Custom-crafted keyframes that handle birth-to-death states, transitions, elastic spring bounces, and subtle breathing scales for perfect, distraction-free loops.
* **Tailwind & CSS Variable Friendly:** Colors and animations can be overridden dynamically by passing styles or variables to their elements!

## 💡 Practical Use Cases

* **Interactive Call-to-Actions (CTAs):** Use the checkmark or sparkles inside buttons to draw subtle attention and delight users upon success.
* **Creative Loading States & Skeletons:** Ditch boring spinners! Use `flowers-falling.svg` or `waypoint.svg` to create soothing, custom loading loops.
* **Unique Hero Section Enhancements:** Add background magic to your landing pages with `sunburst.svg` or floating `stars.svg`.
* **Gamified Success Badges:** Use the bouncing `star.svg` or `check.svg` to celebrate user achievements or completed forms.
* **Custom Cursor & Hover Effects:** Trigger custom micro-bursts dynamically on user actions.

## 🛠️ Integration Guide

### 1. Inline SVG HTML

The best way to preserve full responsiveness and dynamic styling. Copy the contents of the `.svg` file and paste it directly into your HTML:

```html
<div class="icon-container">
  <!-- Paste your SVG code here -->
  <svg>...</svg>
</div>
```

### 2. Using Image Tags

Simple, clean, and supports automatic loop play:

```html
<img src="svg-icons/emojis/star.svg" alt="Animated Star" width="64" height="64" />
```

### 3. React Component Wrap

Easily dynamicize sizes and colors in React/Vite environments:

```jsx
import React from 'react';

export const AnimatedCheck = () => (
  <object type="image/svg+xml" data="svg-icons/emojis/check.svg" className="w-16 h-16">
    Your browser does not support SVGs
  </object>
);
```

### 4. Markdown Embed

Perfect for embedding animated visuals directly inside GitHub `README.md` files:

```markdown
![Animated Star](svg-icons/emojis/star.svg)
```

## 🖌️ Contribution & Aesthetic Guidelines

We welcome community creations! We are not seeking overly complex layouts, but rather designs that are **useful, practical, and cute/pretty to use**.

To generate a new compatible icon, we highly recommend utilizing our custom **`animated-svg-generator`** skill paired with the **Gemini 3.1 Pro** model (which has proven exceptionally skilled with SVG vector syntax and motion geometry).

### 🤖 How to trigger the Generator Skill:

Ask your agent/model:

```text
/animated-svg-generator Create a cute, micro-animated loading heart icon in HSL pastel colors.
```

### ✨ Aesthetic Recommendations:

1. **ViewBox & Sizing:** Keep it standard at `viewBox="0 0 100 100"`.
2. **Outlines:** Use thick outline layers (`3px`–`6px`) with rounded caps and joins (`stroke-linecap="round"` / `stroke-linejoin="round"`).
3. **Transitions:** Implement elastic spring transitions (`cubic-bezier(0.34, 1.56, 0.64, 1)`) for lively, organic responses.
4. **No Color Locks:** Never hardcode colors inside keyframes; always reference parameters (`stroke: var(--secondary-color);`) so variables remain dynamic.
5. **Language:** Ensure all SVG and stylesheet code comments are written in English.
